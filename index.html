O Modelo OSI

[cite_start]O **Modelo de Referência para Interconexão de Sistemas Abertos (OSI)** foi criado pela ISO (Organização Internacional para Padronização) para padronizar a comunicação entre sistemas de diferentes fabricantes, combatendo a heterogeneidade que existia no setor no final dos anos 70[cite: 3, 9, 10]. [cite_start]Ele é um modelo de referência, o que significa que define a arquitetura do sistema e os serviços que cada camada deve fornecer, mas não dita como isso deve ser feito[cite: 40, 41].

[cite_start]A adoção de um modelo em camadas simplifica o projeto, a implementação e a manutenção de uma rede, dividindo a complexidade do problema em partes menores e bem definidas[cite: 23, 25, 26, 28, 29, 31, 33]. Cada camada se concentra em sua função específica e interage apenas com as camadas adjacentes.

7 Camadas do Modelo OSI

O modelo OSI é dividido em sete camadas, cada uma com funções específicas:

    [cite_start]
  • **Camada 7: Aplicação** - Fornece a interface para as aplicações do usuário e provê serviços como transferência de arquivos e acesso a bancos de dados[cite: 352, 649, 650].
    • [cite_start]
  • **Camada 6: Apresentação** - Realiza transformações nos dados, como tradução, compressão e criptografia, para que a sintaxe seja a mesma para ambos os sistemas[cite: 328, 332, 352, 643, 644].
    • [cite_start]
  • **Camada 5: Sessão** - Permite que aplicações em hosts diferentes compartilhem uma sessão, controlando o diálogo e a sincronização entre eles[cite: 320, 323, 352, 629, 631, 634].
  • **Camada 4: Transporte** - É a primeira camada fim-a-fim, responsável por dividir mensagens em pacotes e garantir uma comunicação confiável. [cite_start]Oferece controle de fluxo e multiplexação de conexões[cite: 89, 297, 299, 352, 618, 620].
  • **Camada 3: Rede** - Controla as operações da sub-rede, como roteamento, controle de congestionamento e interconexão de redes. [cite_start]A camada não garante que o pacote chegue ao destino[cite: 285, 286, 287, 288, 352, 597, 598, 605].
  • **Camada 2: Enlace** - Trata do fluxo de dados no enlace entre transmissor e receptor, incluindo controle de fluxo e detecção e correção de erros. [cite_start]Não permite ligação entre redes distintas[cite: 263, 264, 266, 270, 352, 576, 578].
  • **Camada 1: Física** - Trata das características mecânicas, elétricas e de procedimento para conexão física. [cite_start]Transmite uma sequência de bits e não se preocupa com o significado deles[cite: 250, 251, 252, 254, 352, 560, 561].

Serviços e Protocolos

É importante diferenciar **serviço** de **protocolo**. Um serviço é um conjunto de primitivas que uma camada oferece à camada superior adjacente, funcionando como uma interface. [cite_start]Ele define as operações disponíveis, mas não como elas são implementadas[cite: 219, 220].

[cite_start]Já o protocolo é um conjunto de regras que governa o formato e o significado dos quadros, pacotes ou mensagens trocados entre entidades pares (entidades na mesma camada, mas em sistemas diferentes)[cite: 221, 223]. [cite_start]Um protocolo é usado para implementar um serviço e pode ser modificado sem afetar a camada superior, desde que o serviço permaneça o mesmo[cite: 222].

[cite_start]A comunicação entre sistemas ocorre em nível de camadas, ou seja, a camada de Aplicação de um sistema se comunica com a camada de Aplicação de outro, e assim por diante até o nível Físico, onde a comunicação é de fato realizada[cite: 90].

O Modelo TCP/IP

[cite_start]O modelo TCP/IP, base da internet, surgiu de um projeto financiado pela ARPA (Advanced Research Project Agency) em 1969[cite: 677]. [cite_start]Ele não foi desenvolvido como um modelo formal, mas sim a partir da evolução de protocolos que se tornaram padrões militares[cite: 456, 699, 892].

[cite_start]O modelo TCP/IP é mais simples que o OSI, pois possui menos camadas[cite: 455]. [cite_start]Ele combina as camadas de Sessão e Apresentação do OSI na sua camada de Aplicação, e as camadas de Enlace e Física em uma única camada de Acesso à Rede[cite: 452, 454].

Camadas do Modelo TCP/IP

  • **Camada 4: Aplicação** - Este nível trata dos detalhes específicos de cada aplicação. [cite_start]Suporta aplicações de usuário, combinando as funcionalidades de Apresentação, Sessão e Aplicação do OSI[cite: 452]. [cite_start]Exemplos: Telnet, FTP, SMTP, SNMP[cite: 741, 742, 755, 756, 767].
    • [cite_start]
  • **Camada 3: Transporte** - Proporciona um fluxo de dados entre dois hosts (fim-a-fim)[cite: 776]. [cite_start]Usa TCP (confiável, orientado à conexão) e UDP (não-confiável, não orientado à conexão)[cite: 445, 729, 731, 733].
    • [cite_start]
  • **Camada 2: Inter-Rede** - Garante a transmissão de pacotes independente da localização dos hosts[cite: 802]. [cite_start]Funções incluem endereçamento (IP) e roteamento[cite: 720, 721]. [cite_start]Não garante a entrega do pacote[cite: 605].
  • **Camada 1: Interface de Rede** - O modelo não especifica muitos detalhes desta camada. [cite_start]Abrange o driver de dispositivo no SO e a correspondente placa de rede[cite: 826]. [cite_start]Trata dos detalhes de hardware necessários para o interfaceamento físico com a rede[cite: 454, 828, 829].

Encapsulamento

No modelo TCP/IP, quando uma aplicação envia dados, eles passam por cada camada, da mais alta para a mais baixa. [cite_start]Cada camada adiciona sua própria informação de controle (cabeçalho) aos dados da camada superior[cite: 842]. [cite_start]A unidade de dados resultante é chamada de **PDU (Protocol Data Unit)**[cite: 149].

Essa adição de informações é conhecida como **overhead**. [cite_start]É a diferença entre o tamanho inicial e o tamanho final do pacote[cite: 855]. [cite_start]A camada Física, por fim, transforma o pacote em uma sequência de bits para a transmissão[cite: 843]. [cite_start]No destino, o processo é o inverso: cada camada retira o cabeçalho correspondente e entrega os dados para a camada superior[cite: 118].

Comparativo entre os Modelos

Semelhanças Diferenças
    [cite_start]
  • Ambos são baseados em camadas[cite: 442, 505].
    • [cite_start]
  • Ambos possuem camadas de Aplicação, Transporte e Rede (ou Internet) comparáveis[cite: 443, 445].
    • [cite_start]
  • Ambos presumem a utilização de tecnologia de comutação de pacotes[cite: 448].
    [cite_start]
  • O TCP/IP combina as camadas de Sessão e Apresentação do OSI em sua camada de Aplicação[cite: 452].
    • [cite_start]
  • O TCP/IP combina as camadas Física e de Enlace do OSI em uma única camada[cite: 454].
    • [cite_start]
  • O modelo OSI é mais geral, enquanto o TCP/IP é mais específico e orientado a protocolos que se tornaram padrão da internet[cite: 457, 877, 894].

O que é a camada Física do Modelo OSI?
[cite_start]
É a camada responsável pelas características mecânicas e elétricas, e pela transmissão de bits sem se preocupar com o seu significado[cite: 560, 561].
Qual a principal função da camada de Rede no Modelo OSI?
[cite_start]
Controlar as operações da sub-rede, como roteamento e endereçamento lógico (IP)[cite: 597, 598].
O que é um "serviço" no contexto do Modelo OSI?
É o que uma camada oferece à camada imediatamente superior. [cite_start]Define o que é feito, mas não como[cite: 219, 220].
Qual a diferença entre TCP e UDP?
TCP é orientado à conexão e confiável. [cite_start]UDP é não orientado à conexão e não confiável, sendo mais rápido[cite: 731, 733].
O que é o encapsulamento no Modelo TCP/IP?
[cite_start]
É o processo de adicionar informações de controle (cabeçalhos) aos dados em cada camada, à medida que eles descem na pilha de protocolos[cite: 842].
Qual é o objetivo principal do Modelo OSI?
[cite_start]
Padronizar a comunicação para permitir a interconexão de sistemas de diferentes fabricantes (interoperabilidade)[cite: 9].
Qual camada do Modelo OSI é a primeira a ser "fim-a-fim"?
[cite_start]
A camada de Transporte[cite: 297].
O que a camada de Apresentação faz?
[cite_start]
Lida com a sintaxe dos dados, realizando compressão, criptografia e tradução para um formato comum[cite: 328, 332].
O que significa "overhead" em redes?
[cite_start]
É a diferença entre o tamanho inicial dos dados e o tamanho final do pacote após a adição de cabeçalhos por cada camada[cite: 855].
Qual a principal função do protocolo IP?
[cite_start]
Endereçamento lógico dos hosts e roteamento de pacotes na camada Inter-Rede do TCP/IP[cite: 286, 288].
A camada de Sessão controla o uso de "tokens" para qual finalidade?
[cite_start]
Para gerenciar o controle de diálogo em conexões half-duplex, definindo quem tem a vez de transmitir[cite: 323].
O que é a PDU?
[cite_start]
PDU significa Protocol Data Unit (Unidade de Dados de Protocolo), que é a unidade de dados de uma camada, após a adição do cabeçalho de protocolo[cite: 149].
Cite uma semelhança entre OSI e TCP/IP.
[cite_start]
Ambos são baseados em camadas e presumem a tecnologia de comutação de pacotes[cite: 442, 448].
Qual camada do TCP/IP combina as funções de Enlace e Física do OSI?
[cite_start]
A camada de Interface de Rede ou Acesso à Rede[cite: 454].
Qual a principal diferença entre os modelos OSI e TCP/IP?
[cite_start]
O OSI é um modelo teórico genérico, enquanto o TCP/IP foi criado a partir de protocolos práticos e é mais específico para a Internet[cite: 457].
O que é o endereço MAC?
[cite_start]
É um endereço físico de 12 dígitos hexadecimais que identifica uma placa de rede na camada de Enlace[cite: 585, 586].
Qual camada do OSI lida com a detecção e correção de erros?
[cite_start]
A camada de Enlace[cite: 264].
Qual a função da camada de Transporte?
[cite_start]
Ela provê a comunicação fim-a-fim, segmentando e garantindo a entrega confiável dos dados entre os hosts[cite: 297, 299, 618, 620].
Cite um exemplo de protocolo da camada de Aplicação no TCP/IP.
[cite_start]
FTP (File Transfer Protocol), Telnet ou SMTP (Simple Mail Transfer Protocol)[cite: 742, 767].
No Modelo OSI, o que é um SAP (Service Access Point)?
[cite_start]
É uma interface lógica entre duas camadas adjacentes, por onde a camada superior solicita serviços da camada inferior[cite: 235, 236].
Qual camada do TCP/IP não garante que o pacote chegue ao destino?
[cite_start]
A camada Inter-Rede[cite: 605].
No modelo OSI, qual camada é responsável por lidar com a tradução e compressão de dados?
[cite_start]
A camada de Apresentação[cite: 328, 332].
